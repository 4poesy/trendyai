global state management.
 
**Naming conventions across the project:**
-   **camelCase:**
JavaScript variables and functions.

React components and JSX files.
-   **kebab-case:**
Folder and general file names (e.g., `ad-genie-agent.js`).

unless explicitly required by third-party packages.

## **Key Operational Workflows & Enhancements**
 
### **1. Human-in-the-Loop (HIL) Orchestration &
Approval Workflows:**
 
* **Explicit Approval Gates:** For critical outputs or
actions, the workflow *must* pause and require human approval via the Admin
Dashboard. Examples include:
    * **Ad Campaign
Launch/Major Budget Changes:** `AdGenie` drafts, but human must approve before
live.
    * **Website
Design/Code Deployment:** `WebWiz` output requires human review and explicit
deployment trigger.
    * **Long-form
Content/Ebooks:** `ContentCrafter`, `BlogSmith`, `BookSmith` outputs require
human editing/approval before final delivery.
    * **Strategic
Plans:** `StratoBoss` plans require human sign-off.
* **Dashboard Queue:** The Admin Dashboard will feature an
"Approval Queue" or "Tasks Awaiting Review" section.
* **Agent Signaling:** Agents will signal `TrendyAI Core`
(e.g., `puter.emit('agent:approval_needed', { taskId, clientEmail, agentName,
outputId })`) when an output requires human review.
* **Workflow Resumption:** Upon human approval, `TrendyAI
Core` will resume the paused workflow, instructing the next agent to proceed.
 
### **2. Ad Spend Management & Reporting:**
 
* **Ad Spend Tracking:** `PulseTrack` agent must integrate
with Google Ads API, Meta Graph API, TikTok Ads API to **fetch and continuously
monitor client ad spend** against allocated budgets in real-time.
* **Budget Alerts:** Implement automated alerts
(email/dashboard notification) when a campaign approaches predefined thresholds
(e.g., 50%, 80% of budget), and a final alert at 100%.
* **Automated Pausing/Halting:** Provide options for
`AdGenie` or `TrendyAI Core` to automatically pause/halt campaigns when budget
is fully consumed, or flag for immediate human review for budget increases.
* **Comprehensive Reporting:** `PulseTrack` will generate
client-friendly reports detailing ad spend, performance metrics, and ROI,
accessible via the Admin Dashboard for agency staff to share.
 
### **3. Intelligent Client Onboarding & Briefing (for
Traditional/Manual Clients):**
 
* **Enhanced "Add Direct Client" Form:** The Admin
Dashboard's manual client addition form will include a **"Client Briefing
(Natural Language)" text area**.
* **AI-Powered Briefing Parsing:** The `ClientFlow` agent
(or a specialized sub-agent it triggers) will leverage `puter.ai.chat` (e.g.,
with a powerful, accurate model like GPT-4o for complex parsing) to:
    * Parse the
natural language brief.
    * Intelligently
extract structured data: business name, goals, target audience, brand voice,
initial keywords, existing assets (URLs, social handles implicitly mentioned),
specific preferences.
    * Populate or
update the client's record in `puter.kv` with this detailed information.
* **Human Review of Parsed Brief:** The parsed and
structured client data must be presented on the dashboard for **human review
and correction** by agency staff before being finalized and initiating further
automated workflows. This ensures accuracy and avoids AI misinterpretations.
* **Data Validation & Normalization:** Implement
automated validation for client-provided data (e.g., URL formats, email
formats) and normalize data (e.g., convert social media URLs to consistent
handle formats).
 
### **4. Feedback Loop & Agent Continuous Improvement:**
 
* **Direct Feedback Mechanism:** On the Admin Dashboard, for
every agent-generated output (e.g., ad copy, blog post, image), provide a clear
mechanism for agency staff to provide **structured feedback** (e.g., "Too
generic," "Needs more detail," "Perfect,"
"Incorrect data").
* **Feedback Storage:** This feedback will be stored in
`puter.kv` or `puter.fs` alongside the agent's output and linked to the
specific task/agent.
* **Agent Learning:** The `Promptify` agent will regularly
analyze this human feedback to:
    * **Refine
prompts** for specific agents or task types.
    * Identify
patterns of successful/unsuccessful outputs.
    * Potentially
inform **dynamic model selection** (e.g., if a certain model consistently
performs better for "poetic tone" content).
* **Iterative Improvement:** Agents should be designed to
learn and improve their outputs over time based on this feedback.
 
### **5. Creative Asset Versioning:**
 
* For all creative outputs (website code, ad creatives,
long-form content, designs, videos):
    * `puter.fs` will
be used to store multiple versions of these assets.
    * Each iteration
will be saved with a timestamp or version identifier.
    * The Admin
Dashboard will allow agency staff to easily browse, preview, and revert to
previous versions of creative assets.
 
### **6. Proactive Client Communication:**
 
* Beyond initial onboarding, `TrendyAI Core` or specific
agents will trigger **personalized client updates** via `puter.email.send` (or
integrated WhatsApp API if implemented). Examples:
    * "Your ad
campaign has launched!"
    * "Your
monthly SEO report is ready for review."
    * "We need
your approval on the new website design."
* These updates will be configurable and tied to key
workflow milestones.
 
### **7. Resilience & Fallback Strategies:**
 
* **External API Retries:** Implement robust retry
mechanisms with exponential backoff for transient failures when interacting
with external APIs (Google Ads, Meta, TikTok, etc.).
* **Graceful Degradation/Fallback:** For critical services
dependent on external APIs, design agents to:
    * Flag for human
intervention if an API is persistently down.
    * Potentially use
a less preferred (but available) AI model from OpenRouter if a primary one is
unavailable.

## **DETAILED AI AGENTS (25 Total)**
 
Each agent will be a distinct modular unit, responsible for
specific tasks. Their outputs will be stored in `puter.fs` or `puter.kv` and
managed by `TrendyAI Core`.
 
1.  **TrendyAI Core
(Master Orchestrator / Project Manager Agent)**
    * **Role:** The
central brain. Receives client requests (from `trendtacticsdigital.com`
webhooks, emails, admin input). Performs intention recognition and detailed
task decomposition. Assigns tasks to appropriate specialized agents. Monitors
overall project progress and state in `puter.kv`. Integrates outputs from
various agents. Facilitates human review points and collects feedback. Updates
client status on the dashboard. Handles overall error routing and ensures 24/7
operation. Acts as the central collector and dispatcher of all human
interaction data (approvals, feedback, manual inputs, error reports) back into
the AI system for agent learning and workflow adjustment.
    * **Triggers:**
`puter.on('email:incoming')`, `puter.on('webhook:stripe_payment_success')`,
`puter.on('admin:manual_client_add')`, `puter.on('admin:manual_task_trigger')`,
internal agent signals.
    * **Key
Functions:** `processIncomingRequest()`, `decomposeTask()`, `assignAgent()`,
`monitorProgress()`, `integrateOutputs()`, `manageProjectState()`,
`routeToHumanReview()`, `handleAgentFeedback()`, `dispatchNotifications()`.
2.  **ClientFlow
(Smart Client Acquisition & Follow-up)**
    * **Role:**
Manages the entire client acquisition funnel. From initial inquiry (webhook,
email, manual input) to lead qualification and automated follow-ups.
Coordinates with `ONBOARDING AGENT`.
    * **Triggers:**
`puter.on('webhook:client_inquiry')`, `puter.on('email:incoming_lead')`,
`puter.on('admin:add_direct_client')`.
    * **Key
Functions:** `qualifyLead()`, `sendAutomatedFollowup()`,
`assignToOnboardingAgent()`, `parseClientBriefing()`.
3.  **ONBOARDING
AGENT**
    * **Role:** Once a
lead is qualified, this agent initiates the formal onboarding process. Collects
necessary client information (via guided chat, email, or direct input), sets up
their initial project in `puter.kv`, and prepares for service delivery.
    * **Triggers:**
`ClientFlow.assignToOnboardingAgent()`.
    * **Key
Functions:** `collectClientInfo()`, `createClientProject()`,
`assignInitialServices()`, `sendWelcomePack()`.
4.  **WebWiz (Web
Design & Development)**
    * **Role:**
Designs and develops websites based on client briefs. Can create wireframes,
mockups, code (HTML, CSS, JavaScript), and deploy websites. Integrates with CMS
(e.g., WordPress API) for content population if required.
    * **Triggers:**
`TrendyAI Core.assignAgent('web_design')`.
    * **Key
Functions:** `generateWireframes()`, `createMockups()`, `writeCode()`,
`deployWebsite()`, `integrateCMS()`.
5.  **RankRover (SEO
Optimization)**
    * **Role:**
Performs comprehensive SEO audits, keyword research, on-page and technical SEO
recommendations, and generates content briefs for `ContentCrafter`. Monitors
search rankings.
    * **Triggers:**
`TrendyAI Core.assignAgent('seo_optimization')`,
`puter.cron('monthly_seo_audit')`.
    * **Key
Functions:** `performSeoAudit()`, `researchKeywords()`,
`generateSeoRecommendations()`, `createContentBrief()`, `monitorRankings()`.
6.  **StratoBoss
(Strategy Consultant)**
    * **Role:**
Develops overarching digital marketing strategies based on client goals, market
analysis, and competitive intelligence. Creates detailed strategic roadmaps.
    * **Triggers:**
`TrendyAI Core.assignAgent('strategy_consulting')`.
    * **Key
Functions:** `analyzeMarket()`, `developStrategy()`, `createRoadmap()`,
`conductCompetitorAnalysis()`.
7.  **Promptify
(Prompt Engineer)**
    * **Role:**
Dynamically generates and refines prompts for all other AI agents to ensure
optimal outputs from `puter.ai.chat` and `puter.ai.txt2img`. Learns from human
feedback to improve prompt effectiveness.
    * **Triggers:**
`TrendyAI Core.requestPrompt()`, `FeedbackLoop.newFeedback()`.
    * **Key
Functions:** `generatePrompt()`, `refinePrompt()`, `learnFromFeedback()`,
`optimizeModelSelection()`.
8.  **AdGenie (Ad
Copywriter & Campaign Manager)**
    * **Role:** Writes
engaging ad copy, generates creative variations, and manages ad campaigns on
platforms like Google Ads, Meta (Facebook/Instagram), and TikTok. Integrates
with `PixelWitch` for visuals.
    * **Triggers:**
`TrendyAI Core.assignAgent('ad_campaign')`.
    * **Key
Functions:** `writeAdCopy()`, `generateCreatives()`, `setupCampaign()`,
`monitorCampaignPerformance()`, `adjustBids()`.
9.  **MailMage (Email
Marketer)**
    * **Role:** Crafts

## **DEPLOYMENT STEPS (Consolidated)**

This section outlines the consolidated steps for deploying the TrendyAI platform.

1.  **Frontend Deployment:**
    * Deploy the React application (built with Vite/Webpack/Parcel) to a static site hosting service like **Vercel** or **Netlify**, leveraging their generous free tiers.

2.  **Puter.js Integration:**
    * Ensure the `https://js.puter.com/v2/` script tag is correctly included in your `public/index.html` file.

3.  **Puter.js App Configuration:**
    * On the **Puter.com dashboard**, define all **25 agents as separate Puter.js apps/scripts**. Each agent (e.g., `ad-genie.js`, `client-flow.js`) corresponds to a Puter.js application.
    * Configure **`puter.on` event listeners** for external webhooks (e.g., Stripe payment success, client inquiry forms) and incoming emails (e.g., `email:incoming`) within your Puter.js app settings or main orchestrator script.
    * Set up **`puter.cron` schedules** on the Puter.com dashboard for automated recurring tasks (e.g., monthly SEO reports, weekly performance checks).
    * Securely store **API keys** for third-party integrations (Stripe, Google, Meta, TikTok, YouTube, OpenRouter, etc.) as **Puter.js environment variables (secrets)** on the Puter.com dashboard. Your agents' code will then access these securely.

4.  **Database & Storage:**
    * `puter.kv` and `puter.fs` are inherently serverless and managed by Puter.js, requiring no separate deployment steps. Their usage will be defined within your agent and core application code.

5.  **DNS Configuration:**
    * Configure your domain (`trendyAI.com`) to point to your deployed frontend application (on Vercel/Netlify).

6.  **External Webhook Setup:**
    * Configure webhooks in external services (e.g., **Stripe**, your **email service provider**, `trendtacticsdigital.com`'s form submissions) to point to the specific `puter.on` endpoints provided by Puter.js for receiving data and triggering workflows.
